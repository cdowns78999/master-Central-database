// VCC Setup Guide

Install the VRChat Creator Companion (VCC) from vrchat.com/home/download. It manages Unity versions, SDK packages, and project templates automatically. Start every project from VCC — never install SDK manually.

# Install VCC, then create project: $ vcc new avatar MyFirstAvatar $ vcc new world MyFirstWorld

// First World Tutorial

Create a VCC World project. Add a VRCWorld prefab to your scene. Set spawn point, configure respawn height, add a mirror and a chair. Build & test. Your first world can be live in under 30 minutes.

// Scene hierarchy: VRCWorld ├── SpawnPoint (0, 0, 0) ├── Ground (Plane) ├── Mirror (VRCMirror) └── Chair (VRCStation)

// Udon Node Graph

Visual scripting system for VRChat worlds. Drag and drop nodes to create interactions: buttons, teleporters, toggles, synced variables. No coding required — great for beginners.

// Example: Toggle light on interact [OnInteract] → GetComponent(Light) → Branch(isActive) → SetActive(true/false)

// UdonSharp

Write Udon behaviors in C#. Full IDE support, compile-time checking, and access to the VRChat Udon API. The pro approach to world scripting.

using UdonSharp; using UnityEngine; using VRC.SDKBase; public class DoorToggle : UdonSharpBehaviour { public GameObject door; private bool isOpen = false; public override void Interact() { isOpen = !isOpen; door.SetActive(isOpen); } }

// OSC Integration

Open Sound Control protocol for external app communication. Drive avatar parameters from Python, Node.js, or hardware. Used for face tracking, heart rate monitors, and custom controllers.

# Python OSC example from pythonosc import udp_client client = udp_client.SimpleUDPClient( "127.0.0.1", 9000 ) client.send_message( "/avatar/parameters/HeartRate", 72.0 )

// Quest Optimization

Quest worlds: 50K triangles, 2 draw calls max, no realtime lights, baked everything. Use LOD groups, occlusion culling, and texture atlasing to hit performance targets.

// Networking

Synced variables with [UdonSynced]. Manual vs Continuous sync. Ownership transfer with Networking.SetOwner(). Late joiners need OnDeserialization. Network bandwidth is limited — sync only what changes.

[UdonSynced] private int score = 0; public void AddPoint() { Networking.SetOwner( Networking.LocalPlayer, gameObject ); score++; RequestSerialization(); }
