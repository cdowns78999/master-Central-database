The Seven Pillars — What a DSP Actually Needs

Every music DSP — from DistroKid to a local Destin platform — runs on these 7 systems. No shortcuts.

7
Core Systems Architecture
The seven pillars every DSP must build or buy
Technical
📤
1. Content Ingestion Platform
Web app for uploading audio + metadata. Audio validation (format, bitrate, stereo). Metadata intake (ISRC, UPC, genre, credits, lyrics). Cover art validation (3000x3000px min, RGB, no social handles). Async queue processing.
🎵
2. Audio Transcoding Pipeline
Convert master files to every store format. Spotify: FLAC 44.1kHz/24-bit. Apple: 24-bit/96kHz for Digital Masters. Amazon HD: up to 24-bit/192kHz. Stack: FFmpeg (open-source) or AWS MediaConvert (managed). Preserve original quality — stores handle final codec conversion.
📄
3. DDEX Metadata Compliance
ERN 4.3 XML message generation. NewReleaseMessage with MessageHeader, ResourceList, ReleaseList, DealList. ISRC/UPC assignment. Territory/rights management. Free DDEX Implementation Licence + DPID registration.
📡
4. Store Delivery Network
SFTP connections to each store (Spotify, Apple, Amazon, YouTube, TikTok, Deezer, Tidal, Pandora). Each has own endpoint, credentials, folder structure. Batch packaging: XML + audio + art. Delivery confirmation + retry logic.
💰
5. Royalty Collection & Accounting
Ingest sales reports from every store (DDEX DSR + proprietary formats). Match streams to tracks/rights holders. Calculate splits. Currency conversion. Tax withholding (1099/W-8BEN). Generate statements. Process payments.
📊
6. Analytics Dashboard
Near-real-time streaming data. Per-track, per-album, per-territory breakdowns. Revenue tracking + projections. Playlist placement tracking. Demographic data where available.
🔒
7. Rights & Catalog Management
Track ownership and splits. Publishing vs. master rights. Content ID reference files (YouTube). Takedown/update capabilities. Territory-specific availability. Catalog transfer between labels.
