Phase 1 — Manual Foundation

Build Your First Overnight Prompt

1

Create CLAUDE.md

In your project root. This is the file Claude Code reads automatically at the start of every session. Include: a brief project description, your coding standards, which files matter most, and what to never touch. Keep it under 200 instructions — concise and clear beats long and vague. This single file transforms Claude from a generic assistant into your project-specific co-pilot.

# Create it in your project root touch CLAUDE.md # Minimum viable content: ## Project: [Your Project Name] ## Stack: [Your Tech Stack] ## Rules: - Never modify production config files - Always write tests for new functions - Use TypeScript strict mode
2

Create Your First Overnight Command

Create .claude/commands/overnight-review.md — your first slash command. This is a reusable prompt file. When you type /overnight-review in Claude Code, it reads and executes this entire file. Write it like you're giving instructions to a meticulous junior developer at 2AM.

# .claude/commands/overnight-review.md Review the entire codebase. Check for: - Unused imports and dead code - Type errors and type safety issues - Missing or inadequate test coverage - Security vulnerabilities (exposed keys, SQL injection) - Performance anti-patterns Write a detailed report to MORNING_REPORT.md Include: file paths, severity, suggested fixes. Sort by severity (critical first).
3

Create a Daily Brief Command

Create .claude/commands/daily-brief.md. This is your morning intelligence report. It searches for the latest news on topics you care about, summarizes into bullet points, and saves it with today's date. Wake up to curated information instead of scrolling.

# .claude/commands/daily-brief.md Search for the latest news and developments on: - [Topic 1: e.g., AI automation tools] - [Topic 2: e.g., your industry news] - [Topic 3: e.g., competitor updates] Summarize into bullet points. Save to DAILY_BRIEF.md with today's date as header. Max 20 bullets. Only include genuinely new info.
4

Create Modular Rules

Create .claude/rules/ with individual rule files. One per topic: code_conventions.md, testing_rules.md, review_checklist.md. Claude loads these automatically alongside CLAUDE.md. This keeps your instructions modular, organized, and easy to update without touching the main config.

mkdir -p .claude/rules # .claude/rules/code_conventions.md - Use camelCase for variables, PascalCase for components - Max function length: 40 lines - Always destructure props # .claude/rules/testing_rules.md - Every new function needs a unit test - Use describe/it blocks, not test() - Mock external APIs, never hit real endpoints # .claude/rules/review_checklist.md - Check for console.log statements - Verify error handling on all async calls - Ensure no hardcoded secrets
5

Test Manually

Run claude in your terminal. Type /overnight-review. Watch it work. Review the output. Refine the prompt. This step is crucial — you need to see the agent work before you automate it. Iterate on the prompts until the output matches what you'd actually want to wake up to. This is your foundation. Everything else builds on this.

# Launch Claude Code in your project cd /your/project claude # Type your slash command /overnight-review # Review the output in MORNING_REPORT.md # Refine the prompt. Repeat until it's solid. # This manual loop IS the foundation.

Phase 2 — Scheduled Automation

Set It and Forget It

1

The Simple Cron Job

The most straightforward path. One line in your crontab, and Claude Code runs your overnight review at 2AM every single night. No dependencies, no frameworks, no overhead. Just a scheduled command that produces a report while you sleep. Start here before reaching for anything more complex.

# Edit your crontab crontab -e # Add this line (runs at 2AM every night) 0 2 * * * cd /your/project && claude --prompt .claude/commands/overnight-review.md # Save and exit. That's it. # Check it's registered: crontab -l
2

GitHub Actions Schedule

Create .github/workflows/overnight.yml with a cron schedule trigger. Uses anthropics/claude-code-action. Free for public repos. Runs in the cloud, so your machine can be off. The output appears as a GitHub Actions run you can check in the morning. Perfect for teams.

# .github/workflows/overnight.yml name: Overnight Review on: schedule: - cron: '0 2 * * *' # 2AM UTC daily jobs: review: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: anthropics/claude-code-action@v1 with: prompt: "Run overnight review per .claude/commands/overnight-review.md"
3

runCLAUDErun

Free macOS native app. Point it at your project folder, set a schedule, and you're done. No terminal required. It provides a clean GUI for managing your overnight runs, viewing logs, and checking outputs. The easiest entry point for scheduled automation if you're on a Mac.

4

Windows Task Scheduler

For Windows users: open Task Scheduler, create a basic task, set the trigger to 2AM daily, and point the action at a batch script that launches Claude Code. Same concept as cron, different OS. Write a simple .bat file that navigates to your project and runs the command.

:: overnight-review.bat cd C:\Users\you\projects\your-project claude --prompt .claude\commands\overnight-review.md :: Task Scheduler Settings: :: Trigger: Daily, 2:00 AM :: Action: Start a program -> overnight-review.bat :: Conditions: Wake computer to run this task
5

The Morning Delivery

Set up a second scheduled task at 7AM that sends MORNING_REPORT.md to your Slack channel or email. Now you wake up to results — not tasks. The overnight agent works while you sleep, and the morning delivery puts the summary in front of you before your first coffee. Chain the two and the loop is closed.

# Morning delivery cron (7AM) 0 7 * * * cat /your/project/MORNING_REPORT.md | mail -s "Overnight Report" you@email.com # Or send to Slack via webhook: 0 7 * * * curl -X POST -H 'Content-type: application/json' \ --data "{\"text\":\"$(cat /your/project/MORNING_REPORT.md)\"}" \ https://hooks.slack.com/services/YOUR/WEBHOOK/URL

Phase 3 — Autonomous Routines

The Loop That Never Stops

1

The Ralph Loop

The most famous pattern in overnight Claude automation. Create a task.md with your objective and clear success criteria. Then run it in a while loop. Claude iterates — reading, coding, testing, fixing — until the task is done. Geoffrey Huntley ran this pattern for 3 months straight. The key is tight success criteria so the agent knows when to stop.

# task.md — define what "done" looks like Objective: Migrate all API routes from Express to Hono Success Criteria: - All routes in /src/api use Hono syntax - All tests pass - No Express imports remain - Type checking passes with zero errors # The loop while true; do claude --prompt task.md --output-format json 2>/dev/null done
2

continuous-claude

A purpose-built tool for autonomous overnight work. Install it globally, point it at your repo, and run continuous-claude start. It creates branches, writes code, pushes PRs, waits for CI to pass, merges on green, and repeats. One user reported 20 PRs created over a single weekend with zero intervention. The PRs are reviewable, the CI is the guardrail, and you control what merges.

# Install npm install -g continuous-claude # Configure (creates .continuous-claude.json) continuous-claude init # Run continuous-claude start # What it does: # 1. Reads PLAN.md for next task # 2. Creates feature branch # 3. Writes code + tests # 4. Pushes PR # 5. Waits for CI # 6. Merges on green # 7. Picks up next task. Repeat.
3

Multi-Agent Swarm

The advanced tier. Tools like claude-flow or ccswarm let you run multiple Claude agents simultaneously. Each agent gets its own worktree — an isolated copy of your repo — so they can't conflict. A coordinator agent dispatches tasks, and workers execute in parallel. 12 agents can rebuild an entire frontend overnight, producing 10,000+ lines of coordinated changes.

# claude-flow: orchestrated multi-agent npx claude-flow start "Rebuild the dashboard with 6 agents" # ccswarm: parallel worktree agents npx ccswarm --agents 12 --task tasks/ # Each agent gets: # - Its own git worktree (isolated branch) # - A specific task from the coordinator # - Access to shared context via SHARED_TASK_NOTES.md # - Automatic PR creation when done
4

The Heartbeat Pattern

The agent wakes every hour. Checks a task queue (a simple file or database). Executes the top-priority task. Reports status. Goes back to sleep. Repeat. Perfect for business operations that need continuous monitoring — like checking for new support tickets, scanning for price changes, or processing incoming data. Low overhead, high reliability.

# Heartbeat: run every hour 0 * * * * cd /your/project && claude --prompt .claude/commands/heartbeat.md # .claude/commands/heartbeat.md 1. Read TASK_QUEUE.md 2. Execute the top uncompleted task 3. Mark it as done with timestamp 4. Log results to HEARTBEAT_LOG.md 5. If errors, add to ERRORS.md with context

The Compound Effect

How Each Phase Builds on the Last

Phase 1 — Manual
You Learn What Works

Refine your prompts. Build muscle memory. Discover which instructions produce great output and which need iteration. Every manual run teaches you something about how to communicate with the agent. This phase is where you build the intuition that makes everything else work.

Phase 2 — Scheduled
Consistency Compounds

Every night, something runs. Results accumulate. Your codebase gets cleaner, your briefs get sharper, your morning workflow gets faster. The power isn't in any single run — it's in the compounding effect of running every single night without fail. Consistency is the multiplier.

Phase 3 — Autonomous
The Machine Works for You

You design the objective. The agent executes. PRs appear in your inbox. Reports land in your Slack. Code gets written, tested, and merged while you're asleep. You shift from doing the work to designing the systems that do the work. That's the real leverage.

Phase 4 — Ecosystem
The Compound Machine

Multiple routines running in parallel. Morning brief + overnight code review + research aggregation + content generation. Each routine feeds the others. The research agent discovers a new library; the code agent integrates it; the review agent checks it; the brief agent reports it. An ecosystem, not a tool.

The best overnight workflows are tightly bounded, have clear success criteria, and use existing review mechanisms (PRs, CI) as guardrails. The technology works — the challenge is designing the right handoff.
